## User Story

Given `server.js` exists, `npm install` has been run, and mkcert certificates are present,
When the developer runs `node server.js`,
Then the server starts without errors, serves files from `public/` over HTTPS, and accepts Socket.IO connections on port 3000.

## Acceptance Criteria

1. `server.js` exists at the project root and is the single entrypoint for the application.
2. Running `node server.js` exits with no uncaught exceptions and does not immediately terminate.
3. Express is configured to serve all files in `public/` as static assets.
4. `GET http://localhost:3000/` returns HTTP 200 (or 304) with a file from `public/`.
5. The HTTP server and Socket.IO share the same port (3000) — no separate WS port.
6. A Socket.IO client can connect to `http://localhost:3000` and receive the `connect` event.
7. The server logs a message to the terminal when it is ready (e.g., "Server listening on port 3000").
8. The server handles the `connection` Socket.IO event and logs when a client connects.
9. The server handles the `disconnect` Socket.IO event and logs when a client disconnects.
10. No hardcoded production config — port 3000 should be defined as a constant or read from `process.env.PORT` with a fallback.
11. The server uses HTTPS (`https.createServer({ key, cert }, app)`) not HTTP. Certificate and key are loaded from `cert.pem` and `key.pem` at the project root (generated by mkcert).
12. Socket.IO is configured with `maxHttpBufferSize: 10e6` (10 MB) to accommodate full-resolution PNG frames (~3–5 MB each).
13. At startup, if `cert.pem` or `key.pem` are not found, the server prints clear setup instructions and exits with a non-zero code:
    ```
    ✗ HTTPS certificates not found.
    Run the following commands to set up mkcert:
      1. Install mkcert: https://github.com/FiloSottile/mkcert#installation
      2. mkcert -install
      3. mkcert -key-file key.pem -cert-file cert.pem 192.168.20.28 localhost 127.0.0.1
      4. On your iPhone: Settings → General → VPN & Device Management → trust the mkcert CA
    Then restart the server.
    ```

## Gotchas & Edge Cases

- `socket.io` must be attached to the `https.Server` instance, not the Express `app` directly — otherwise WebSocket upgrades will not work.
- `express.static` is order-sensitive: register it before any catch-all route, or `public/` files will be shadowed.
- If `public/` is empty, `GET /` returns 404. A placeholder `index.html` should exist so the acceptance criterion can be verified before the dashboard is built.
- `maxHttpBufferSize` must be passed as an option to the `Server` constructor: `new Server(httpsServer, { maxHttpBufferSize: 10e6 })`.
- Socket.IO 4.x enables CORS permissively by default in development — fine for local use, but note this if the project is ever exposed beyond the LAN.
- Avoid calling `app.listen()` — use `server.listen()` on the `https.Server` so Socket.IO can intercept upgrades on the same port.
- The cert is generated for the specific local IP `192.168.20.28`. If the PC's IP changes (DHCP), the cert must be regenerated. Document this in the README.
